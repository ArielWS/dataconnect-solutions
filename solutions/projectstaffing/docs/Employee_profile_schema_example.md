**Field descriptions**

It can be noticed that there's a list of fields: skills,about_me,responsibilities,job_title that is duplicated with
skills_v2,about_me_v2,responsibilities_v2,job_title_v2. That was necessary due to a technical reason in order to support
2 types of field analysis for the same type of fields. Since azure search doesn't support multiple analyzers per field,
the fields of interests had to be duplicated.

_key_lemma_ fields contain the lemma extracted from the associated _key_ fields. (about_me,skills_responsibilities)

_curated_all_ field contains all the information extracted from skills,about_me,responsibilities.

_de_domain fields: contain terms from that employee profile that belong to a domain specific vocabulary.

_de_domain_mapping fields: contain terms from that employee profile that belong to a domain specific vocabulary and have
a corespondent related terms list.
